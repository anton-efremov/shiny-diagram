## Required commands
#### Fully implemented commands already (declaration in View, implementation in Controller)

- **class.directStyle.property.set**
- **class.duplicate**
- **class.delete**

#### Declared commands already (not implemented in Controller yet)

- **class.directStyle.clear** — needed by every "garbage collection" effect and by Save style
- **style.definition.create** — needed by Save style (see shape extension below)
- **style.definition.delete** — needed by Delete style; translate must cascade removal of all applications of the styleDef, since View cannot enumerate applications
- **style.definition.name.set** — needed by StyleNameEditor; renames identity (StyleDefId derived from name), Controller rewrites all application references
- **style.definition.property.set** — needed by StyleEditPane palette ("update S for all wearers")
- **style.application.create** — superseded by class.appliedStyle.set if adopted (see below)
- **style.application.delete** — superseded by class.appliedStyle.set if adopted; keyed by StyleApplicationId, which is absent from the view schema
- **style.application.styleDefinition.set** — superseded by class.appliedStyle.set if adopted; same StyleApplicationId gap

#### Not declared commands

- **class.directStyle.set** — writes a full `StyleProperties` direct style in one command. Required by "create a direct style from S with one property changed" (journeys A, C, D): composing it from several `property.set` commands in one transaction breaks against one frozen snapshot — each would synthesize its own `style Foo …` insert.
- **style.definition.create — shape extension: `applyToClassIds: readonly ClassId[]`** — Save style must create the def and apply it in one transaction, but the transaction limitation forbids targeting the new styleDefId in a later command. Precedent: `namespace.create.initialClassIds`.
- **class.appliedStyle.set** — `{ classId, styleDefId | null }`: sets/replaces/removes the class's style application keyed by class, not by StyleApplicationId. ClassView carries `appliedStyleId`, not the application edge id, so View cannot construct the declared `style.application.*` commands. Alternative: expose `StyleApplicationId` on ClassView and implement the three declared commands. Decision needed; class-keyed is the smaller surface given one applied style per class.
